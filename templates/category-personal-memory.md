# Category-Specific Analysis: Personal / Local Memory System

Use this section for personal memory systems, local second brains, agent-maintained personal brains, and personal knowledge bases.

## Human vs Agent Orientation

- Is the system primarily human-readable, agent-facing, or both?
- Can people browse and edit the memory directly?
- Can agents write memories without breaking human readability?
- Does the system produce durable knowledge or just retrieval snippets?

## Knowledge Structure

- Does it use notes, documents, timelines, entities, tags, embeddings, or a knowledge graph?
- How are links between memories represented?
- Does it distinguish raw data, extracted facts, summaries, and canonical knowledge?
- How does it handle contradictions, stale facts, and duplicates?

## Ingestion and Maintenance

- What sources can it ingest?
- Does it automatically summarize, tag, link, or compile memories?
- Can users approve or reject memory updates?
- Does it support forgetting, decay, archival, or version history?

## Locality and Ownership

- Where is the memory stored?
- Can users export or back up the full memory?
- Can the memory be used by multiple agents or apps?
- Is the system tied to one UI or one agent harness?

## Personal Memory Comparison Axes

| Axis | Key question |
|---|---|
| Human readability | Can users directly inspect and maintain the memory? |
| Agent writability | Can agents safely create and update memories? |
| Knowledge structure | Does it use files, graph, embeddings, timeline, or another model? |
| Memory hygiene | How does it handle stale, conflicting, or duplicate memory? |
| Ownership | Can users export and move the memory? |
| Cross-agent use | Can multiple agents use the same personal memory? |
