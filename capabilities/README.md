# Capabilities

Capability pages define the questions used to evaluate second-brain solutions. The detailed per-solution scoring lives in [../comparisons/capability-matrix.md](../comparisons/capability-matrix.md).

| Capability | Key question |
|---|---|
| [Data Capture](data-capture.md) | How does raw context enter the system? |
| [Auto Organization](auto-organization.md) | Does raw input become structured, retrievable knowledge? |
| [Dreaming / Consolidation](dreaming-consolidation.md) | Is there a periodic cleanup, reflection, or maintenance loop? |
| [Agent Access](agent-access.md) | Can external agents search, read, write, or update memory? |
| [Team / Workspace Boundaries](team-workspace-boundaries.md) | Can users separate personal, project, team, and organization contexts? |
| [Privacy / Control](privacy-control.md) | Can users inspect, edit, delete, export, self-host, or audit memory? |
| [Setup / Operations](setup-operations.md) | What does it take to install, verify, and keep the workflow healthy? |
