# Category-Specific Analysis: Team / Enterprise Knowledge Memory

Use this section for organizational knowledge systems, enterprise search, meeting memory, team AI assistants, and internal RAG systems.

## Organizational Scope

- What sources are indexed: docs, Slack, email, meetings, tickets, CRM, code, drives?
- Is memory scoped by team, department, workspace, or organization?
- Does the system distinguish individual memory from shared team memory?
- Does it support cross-functional or company-wide knowledge?

## Permission-Aware Retrieval

- Does retrieval respect source permissions?
- Are admin controls available?
- Are search, answer, and memory access auditable?
- Can sensitive data be excluded or scoped?

## Human Search vs Agent Memory

- Is the system mainly a human-facing search/answer tool?
- Can agents use it as durable memory?
- Can agents write new memories, summaries, decisions, or facts?
- Are memories structured enough for workflows, or mostly unstructured search?

## Governance and Lifecycle

- How are stale documents, outdated decisions, and conflicting sources handled?
- Does it support retention policies, compliance, or legal hold?
- Can teams curate canonical knowledge?
- Can users trace answers back to sources?

## Team Knowledge Comparison Axes

| Axis | Key question |
|---|---|
| Source coverage | Which team data sources are included? |
| Permission awareness | Does retrieval respect enterprise access controls? |
| Shared memory scope | Is memory team, department, or org-level? |
| Agent usability | Can agents read and write usable memory? |
| Governance | Can admins manage lifecycle, compliance, and audit? |
| Source traceability | Can answers and memories be traced back to evidence? |
