# Category-Specific Analysis: Frontier AI Platform

Use this section for platform-native memory inside major AI products such as ChatGPT, Claude, Gemini, Perplexity, and Grok.

## Platform Memory Model

- Is memory account-level, workspace-level, project-level, chat-level, or mixed?
- Is memory a visible product feature, hidden personalization layer, or both?
- How is memory different from chat history, custom instructions, project files, or training data?
- Does the platform distinguish temporary context from durable memory?

## User Visibility and Controls

- Can users inspect the exact memories being used?
- Can users edit or delete individual memories?
- Can users disable memory globally or per conversation/project?
- Are there admin controls for team or enterprise plans?
- Are memory changes auditable?

## Cross-Surface Behavior

- Does memory work across web, desktop, mobile, API, and agent surfaces?
- Does memory transfer across different models inside the same platform?
- Does memory apply to custom bots, projects, gems, GPTs, or equivalent product surfaces?
- Can memory be scoped away from sensitive projects?

## External Access and Portability

- Is memory accessible through an API, export, or integration?
- Can external agents read or write the platform memory?
- Can memory be migrated to another system?
- What lock-in does the platform create?

## Frontier Platform Comparison Axes

| Axis | Key question |
|---|---|
| Memory visibility | Can users see what the platform remembers? |
| Product scope | Is memory global, project-level, workspace-level, or chat-level? |
| Platform lock-in | Can memory leave the product? |
| API access | Can developers or agents access memory directly? |
| Enterprise controls | Can teams govern memory behavior? |
| Personalization boundary | Is memory clearly separated from chat history and training? |
